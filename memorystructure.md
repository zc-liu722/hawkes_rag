# 一套基于使用动力学的 Agent 记忆机制

## 核心思想

记忆的重要性不靠 LLM 标注,而是从"被使用"这一行为中自然涌现。白天用一个零 token 的活跃度动力学(λ)做即时召回与遗忘筛选,夜间用一次低频 dreaming 只对高价值幸存者做巩固。LLM 几乎不参与日常回路,只在每天一次的离线巩固时进场。

## 一、存储介质:单一介质,turn 粒度

所有记忆是统一的一个池子。记忆单元是一个 turn 或 session 的原始片段,带 embedding,永不拆解成命题。每条片段附带一个活跃度标量 λ ∈ (0,1]。冷记忆和热记忆不是两个库,而是同一个池子里 λ 高低的连续光谱——区别只在召回路径,不在存储。

写入 = 片段追加入池 + λ 初始化为 1,全程零 LLM。

## 二、活跃度动力学(白天,零 token)

**召回打分。** 对查询 q 与片段 m,召回分数为:

score = cos(q, m) · [μ + (1−μ)·λ⁻]

其中 λ⁻ 是该片段被调用前的活跃度,μ 是噪音系数。μ 由当前检索池中 λ 分布的归一化香农熵决定:分布越集中(熵低,系统已长出明确偏好),μ 越大,让语义相似度重新主导;分布越平摊(熵高,系统尚无可信偏好),μ 越小,让活跃度有发言权。这等于让系统自己判断"我的使用历史现在可信吗"。

**衰减。** 片段创建后或两次成功调用之间,λ 随时间指数自然衰减。衰减系数按事实类型分档:易变信息衰减快,稳定信息衰减慢,身份类信息几乎不衰减。

**激励。** 被成功调用(定义见第四节)后,λ⁺ = λ⁻ + (1−λ⁻)·score。其中 (1−λ⁻) 是"剩余可激励空间",它本身就是富者愈富的内生阻尼器:λ 越高,单次激励增量越小,系统自动收敛、不会爆炸。

## 三、三种召回源:层级兜底

三者不是并联,而是按成本递进的层级兜底,大多数查询在前两层就解决,根本不触发冷召回。

**1. 天然上下文(MEMORY.md)——常驻,不检索。** dreaming 提炼出的高价值长期记忆,每次对话直接挂在 system prompt 顶部。它是最重要信息的载体,因为只装幸存者所以体量小,常驻而不撑爆上下文。最关键的记忆根本不需要"召回"这个动作,它一直在场。

**2. 热召回——每轮隐式触发,零 token。** 每个 user turn,用 score 公式在池中取候选,经阈值门控后注入。这是默认路径。

**3. 冷召回——条件触发,零 token。** 仅当热召回 top-1 的 score 低于阈值(说明高 λ 区没有好答案)时,绕过 λ、在全池(含已沉降的低 λ 片段)跑一次纯语义检索。这条路径专门闭合"刻意找旧信息"的需求:旧信息从未被删除,只是被 λ 排到了后面,绕过 λ 就能找回。显式工具调用作为补充保留,隐式 fallback 是主力。

## 四、激活动作:被采纳才激活,不是被召回就激活

这是整个动力学的引擎,也是堵住 learning leak 的关键。

"被召回"(进了候选、过了阈值、塞进上下文)**不**触发激励——否则语义相似但已过时的片段会每次被召回、每次被强化、永不衰减。

只有"被采纳"(该片段实际进入了模型的最终回答)才触发 λ⁺ 激励。采纳判定不引入新的 LLM 调用,可用回答文本与片段的语义/n-gram 重叠启发式判断,或搭主 agent 那次生成的便车让它自报用了哪几条编号。塞进上下文的片段集合天然就是激活候选集合,被采纳的子集即激活集合,无需任何额外的"返回编号"机制。

## 五、注入策略:有效且不污染上下文

**分区放置。** MEMORY.md 放 system prompt 顶部(权威、稳定);热召回片段放在当前 turn 正上方,用明确标记包裹,让模型知道这是检索来的、未必全相关、可以不用;冷召回只在触发时临时注入,用完即走,不进对话历史。

**阈值是抗污染总闸。** 低于注入阈值 τ 的片段一律丢弃。这是上下文污染的总开关——没有它,系统会在"其实没有相关记忆"时硬塞噪音。而 score 公式天然把"语义又不像、又早被遗忘"的片段压到阈值下,所以这套机制本身就是一个抗污染过滤器,这是它相对纯 RAG 的固有优势。

**注入量随 score 分布自适应,而非固定 k。** 最高分断崖领先 → 只塞 1-2 条(高置信,冗余反而稀释);分数平缓 → 多塞几条让模型自己挑;全部低于 τ → 一条不塞。"该沉默时能沉默"是这套机制区别于 RAG(永远塞 top-k)的能力。

**附带元数据。** 每条片段标注时间、距今多久、当前 λ,让模型在生成时自行判断时效性,把时间感知的一部分交给模型而非全压给机制。

## 六、Dreaming:夜间巩固,LLM 唯一进场点

每天触发一次,是全系统唯一消耗 LLM 的地方,低频、批量、可缓存、成本可预测。它做三件事,且只对高价值区操作:

**1. 抽取。** 只扫 λ 高且被反复采纳的幸存片段,LLM 把它们提炼写入 MEMORY.md。核心约束:不是不抽取,而是让 λ 动力学先做一整天的零成本筛选,只有挺过衰减、被反复采纳的记忆才配消耗 token 去抽取。token 花在被验证过的信息上。

**2. 固化。** 已写入 MEMORY.md 的片段语义已被天然上下文承载,其原片段可沉降(λ 降低),原文留池作审计,不物理删除。

**3. 矛盾消解。** 在 fact level 上对 MEMORY.md 做写入、修改、删除。新晋升内容与已有内容冲突时(如"在东京"与新的高 λ 片段"搬回伦敦"),在此一次性裁决并改写。矛盾消解放在这里而非实时:低频、批量、有完整上下文,既避免了实时调用的 token 消耗,也降低了幻觉概率。

## 七、架构原则:无子 agent

系统只有一个 agent(主 agent)。记忆不是第二个 agent,而是夹在主 agent 与存储之间的一个无状态确定性中间件:拦截 query → 打分排序 → 阈值过滤 → 注入 → 回收采纳信号更新 λ。dreaming 作为离线任务异步挂在旁边。

判据:一个职责需要独立 agent,当且仅当它需要自主、多步、有状态的推理。本系统中召回是单步计算、激活是集合操作、相关性判断已被 score 公式吸收、矛盾消解已离线化——没有一项满足,故不引入子 agent。引入子 agent 只会带来 agent 间通信、状态同步、额外 LLM 调用等复杂度,并亲手破坏"日常零 token"这一核心优势。

## 八、差异化定位

相对主流系统:Mem0/Zep 实时抽取记忆(高频 token、有抽取无演化或演化重型),本机制延迟到夜间且只抽幸存者;Generative Agents 有 reflection 但需全量反思且 importance 静态,本机制用 λ 先做零成本筛选、importance 由使用动态涌现;Letta 处处用 LLM 管记忆所以昂贵,本机制日常零 LLM、仅 dreaming 一天进场一次。

一句话:**一个单一介质、turn 粒度、写入零 LLM 的记忆系统;白天靠 λ 使用动力学完成零 token 的召回与遗忘筛选,夜间用一次低频 dreaming 只对高 λ 幸存者做巩固与矛盾消解;三种召回源层级兜底,激活信号取"被采纳"而非"被召回"以根除 learning leak。**
